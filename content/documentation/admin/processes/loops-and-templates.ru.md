Дополнение к разделу [«Цикл»](./overview/#цикл) и описание элементов **«Шаблон»** и **«Таймер»**.

## Режимы цикла

Элемент **«Цикл»** поддерживает два режима (поле «Режим цикла» в боковой панели):

| Режим | Поле в интерфейсе | Поведение |
| ----- | ----------------- | --------- |
| Фиксированное число | «Количество итераций» (1–10000) | Тело цикла выполняется заданное число раз |
| По коллекции | «Шаблон коллекции» | При входе вычисляется Go-шаблон; результат должен быть **JSON-массивом**. Число итераций равно длине массива |

Примеры «Шаблона коллекции»:

```text
{{ toJSON .store.branch_tags }}
{{ toJSON .store.notification.engagements }}
```

- **Пустой массив** — 0 итераций, сразу переход на **«Выход из цикла»** без ошибки.
- **Не массив** или **более 10000 элементов** — запуск завершается с ошибкой.

### Исходящие связи

Из элемента «Цикл» должны выходить **две** связи:

1. **«Тело цикла»** (нижний порт) — ветка, выполняемая на каждой итерации.
1. **«Выход из цикла»** (правый порт) — продолжение после последней итерации.

В боковой панели цикла укажите, какой из двух исходящих элементов соответствует **выходу из цикла**.

## Текущий элемент итерации

Внутри тела цикла в шаблонах (правила хранилища, условия шлюзов, элемент «Шаблон», «Шаблон коллекции» вложенного цикла) доступны значения **текущей итерации** через хранилище:

| Фрагмент шаблона | Описание |
| ---------------- | -------- |
| `{{ .store._loop.item }}` | Текущий элемент массива (режим «По коллекции») или номер итерации (режим «Фиксированное число») |
| `{{ .store._loop.index }}` | Номер итерации, начиная с 0 |
| `{{ .store._loop.parent }}` | Контекст **внешнего** цикла при вложенности |

Чтобы записать объект целиком в JSON, в поле «Источник» операции «Записать JSON» используйте, например: `{{ .store._loop.item | toJson }}`.

{{< alert level="info" >}}
В **эксклюзивном шлюзе** и **элементе «Шаблон»** контекст итерации подставляется автоматически. Для задач с **внешним HTTP-действием** внутри цикла правила записи в хранилище должны быть настроены на том же шаге или на задаче «Обновление хранилища процесса», иначе в шаблоне может не быть актуального элемента цикла.
{{< /alert >}}

## Вложенные циклы

Допускается цикл внутри цикла (например, обход engagements внутри каждой branch tag). У внутреннего цикла свой `{{ .store._loop.item }}`; данные внешнего цикла доступны через `{{ .store._loop.parent }}`.

## Эксклюзивный шлюз в цикле

Чтобы отобрать итерации по полю элемента, задайте **условие с шаблоном**, а не только проверку статуса задачи:

```text
{{ .store._loop.item.branch }}  equals  {{ .store.current_branch_tag }}
```

Рекомендуемый порядок исходящих веток:

1. **Первая ветка** — условие по полю (например, тип сканирования `cve`).
1. **Вторая ветка** — выражение `true` (ветка «иначе» — пропустить итерацию или общий путь).

Условия по полям хранилища проверяются **раньше** ветки с `true`, иначе «иначе» сработает слишком рано.

## Элемент «Шаблон»

Элемент **«Шаблон»** формирует текст по Go-шаблону и **записывает строку** в хранилище. Он **не** отправляет сообщения во внешние системы — для отправки используйте следующую **задачу** (webhook, мессенджер и т. д.), которая читает записанный ключ.

| Поле | Описание |
| ---- | -------- |
| Тело шаблона | Go-шаблон (Markdown, обычный текст, HTML или JSON как текст) |
| Ключ в хранилище | Куда записать результат (обязательно), например `rendered_message` |
| Подсказка формата | Подсказка в интерфейсе (`text`, `markdown`, `html`, `json`); на результат не влияет |

В шаблоне доступны **хранилище**, **параметры процесса**, данные **сущности**. Параметры процесса в формате JSON-массива (например список типов сканирования) можно обходить в цикле шаблона.

Типовая цепочка на схеме:

```text
… циклы и обновление хранилища … → «Шаблон» → задача отправки
```

Максимальный размер **тела шаблона** — **1 МБ**; суммарный размер хранилища — тоже **1 МБ**.

### Пример фрагмента тела

```text
{{ $n := .store.notification }}{{ range $.process.scan_type_tags }}{{ $tag := . }}{{ range $n.branches }}
### {{ $n.module_name }} — {{ .name }} ({{ default "" (index . "image_tag") }})
...
{{ end }}{{ end }}
```

Поля, используемые в шаблоне (например `image_tag` у ветки), должны быть **заполнены правилами хранилища** на предыдущих шагах.

## Элемент «Таймер»

Элемент **«Таймер»** приостанавливает процесс до наступления времени:

| Режим | Поведение |
| ----- | --------- |
| Задержка после входа в элемент | Пауза на заданное число секунд (от 1 до 1209600) |
| Произвольное расписание | Первое срабатывание не ранее указанной даты или интервала; далее по правилу (еженедельно, ежемесячно, каждые N дней) |

Пока таймер не сработал, запуск может быть в статусе **«Ожидание»**. На экране «Запуск процесса» показывается ориентировочное время продолжения.

## Пример сценария

Схема отчёта CSE (DefectDojo → хранилище → текст → мессенджер):

1. Загрузка данных в хранилище (внешние действия или тестовая загрузка).
1. Цикл «По коллекции» по тегам веток → инициализация ветки → внутренний цикл по engagements → шлюзы по типу сканирования → накопление в `notification.branches`.
1. Элемент «Шаблон» → ключ `rendered_message`.
1. Задача отправки сообщения.
