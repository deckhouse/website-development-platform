## Новые возможности

### Источники данных

- Добавлены **подсказки при настройке параметров** источников данных — [параметры источников данных](../../admin/datasources/overview/#параметры-источников-данных).
- Добавлен **импорт параметров** — по JSON, YAML или OpenAPI-спецификации данных можно автоматически создать параметры ресурса и заполнить правила обновления источника данных — [импорт из спецификации](../../admin/datasources/overview/#импорт-из-спецификации).

### Виджеты

Добавлены новые виджеты:

- **Просмотр репозитория** — для просмотра структуры и содержимого файлов в репозиториях — [описание виджета](../../admin/widgets/types/#просмотр-репозитория).
- **Jenkins. Пайплайны** — для управления сборками в Jenkins — [описание виджета](../../admin/widgets/types/#jenkins-пайплайны).
- **Jira. Задачи** — для просмотра задач в Jira — [описание виджета](../../admin/widgets/types/#jira-задачи).
- **Bitbucket. Pull Requests** — для просмотра и управления Pull Requests в Bitbucket — [описание виджета](../../admin/widgets/types/#bitbucket-pull-requests).
- **Vault. Секреты** — для просмотра секретов в HashiCorp Vault или Deckhouse Stronghold — [описание виджета](../../admin/widgets/types/#vault-секреты).

### Процессы и действия

- Добавлен механизм обмена данными между действиями процесса: результаты выполнения действия можно записать в хранилище и использовать в последующих действиях — [хранилище процесса](../../admin/processes/#хранилище-процесса).
- Добавлена возможность отмены запущенных действий и процессов.

### Команды

Добавлены правила фильтрации при синхронизации команд — [фильтрация групп при синхронизации](../../user/teams-users/#фильтрация-групп-при-синхронизации).

### Каталог

Добавлена возможность выгрузки сущностей нескольких ресурсов в CSV-файл — [выгрузка сущностей в CSV](../../user/catalog/#выгрузка-сущностей-в-csv).

### Ролевая модель

- Добавлено глобальное разрешение `edit:team-filter-rules` для настройки правил фильтрации при синхронизации команд — [глобальные разрешения](../../admin/security/rbac/#глобальные-разрешения).
- Добавлено глобальное разрешение `edit:user-access-credentials-types` для управления типами учётных данных — [глобальные разрешения](../../admin/security/rbac/#глобальные-разрешения).

### Параметры

Добавлен новый тип параметра **Enum** — [тип параметра Enum](../../user/properties/#enum).

### Учётные данные

Добавлена возможность хранения пользовательских учётных данных в **HashiCorp Vault** или **Deckhouse Stronghold** — [хранение в HashiCorp Vault](../../admin/security/credentials/#хранение-в-hashicorp-vault).

## Улучшения интерфейса

- Пункты верхнего и боковых меню, а также карточки в разделе «Самообслуживание» теперь можно открыть в новой вкладке: нажмите средней кнопкой мыши или, удерживая Ctrl (Windows/Linux) либо Cmd (macOS), нажмите левой кнопкой мыши.
- Улучшена форма редактирования источников данных.

## Исправления

- Решена проблема с отображением команды-владельца в карточках раздела «Самообслуживание».
- Исправлено удаление линий в редакторе процессов.

## Предупреждение об удалении

В релизе **1.4.0** будут удалены следующие механизмы:

- **Источники данных для конкретных ресурсов Kubernetes**: `KubernetesNamespaces`, `KubernetesDeployments`, `KubernetesIngresses`, `KubernetesPods` — для замены рекомендуется использовать общий источник данных [KubernetesResources](../../admin/datasources/types/#kubernetesresources).
